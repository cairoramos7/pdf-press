# Contributing to PDF Press

## Commit Messages

We follow the [Conventional Commits v1.0.0](https://www.conventionalcommits.org/en/v1.0.0/) standard.

**Format**:
```
<type>: <description>
```

**Important**: We do **not** use scopes/namespaces in this project.

### Examples
- `feat: add pdf compression logic`
- `fix: resolve crash on startup`
- `docs: update readme instructions`

### Types
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `style`: Formatting, missing semi-colons, etc,
- `refactor`: Code refactoring
- `test`: Adding tests
- `chore`: Maintenance tasks
